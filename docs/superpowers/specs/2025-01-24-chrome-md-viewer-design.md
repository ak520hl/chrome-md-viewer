# Chrome Markdown Viewer 设计文档

## 概述

Chrome Markdown Viewer 是一个谷歌浏览器插件，用于浏览本地Markdown文档库。用户可以在浏览器中直接打开.md文件，插件会显示文件所在目录的所有文档，并提供GitHub风格的渲染效果和目录导航功能。

## 核心需求

1. **本地文档浏览器** - 浏览本地Markdown文档库（笔记、文档集合）
2. **直接在浏览器中打开** - 通过Chrome Extension拦截.md文件打开请求
3. **GitHub风格渲染** - 代码高亮、表格、任务列表等
4. **可折叠侧边栏** - 默认展开，可点击按钮折叠隐藏
5. **右侧固定目录** - 显示当前文件的标题结构
6. **File System Access API** - 使用现代浏览器API读取本地文件

## 技术栈

- **Chrome Extension** - 浏览器插件框架
- **marked.js** - Markdown解析器
- **highlight.js** - 代码语法高亮
- **GitHub Markdown CSS** - GitHub风格样式
- **File System Access API** - 本地文件系统访问

### 核心组件

1. **Background Script**
   - 处理扩展生命周期
   - 管理文件系统权限
   - 存储最近打开的目录

2. **Viewer Page**
   - 主界面，包含三个面板
   - 管理文件选择和渲染状态
   - 处理用户交互

3. **File System Manager**
   - 封装File System Access API
   - 读取目录和文件
   - 监听文件变化

4. **Markdown Parser**
   - 使用marked.js解析Markdown
   - 提取标题生成TOC
   - 渲染为HTML

5. **UI Components**
   - Sidebar：文件列表和目录树
   - Content：Markdown渲染区域
   - TOC：目录导航面板

## 组件设计

### 1. Sidebar组件

```
┌─────────────────────────┐
│ [📁 文件列表] [📑 目录]  │  ← Tab切换
├─────────────────────────┤
│ ▸ 当前目录               │
│   📄 readme.md          │
│   📄 guide.md           │
│   📁 子目录              │
│     📄 api.md           │

│                         │

**功能**：

- Tab2：显示当前打开文件的目录结构（标题层级）

- 支持折叠/展开目录

### 2. Content组件

```

├─────────────────────────────────────┤
│                                     │

│                                     │

```

- 渲染Markdown为HTML
- 代码语法高亮

- 平滑滚动

### 3. TOC组件

```
┌─────────────────────┐
│ 📑 目录             │
├─────────────────────┤
│ ▸ 标题1             │
│   ▸ 标题2           │
│     ▸ 标题3         │

│ ▸ 标题1             │
│   ▸ 标题2           │

**功能**：
- 提取Markdown中的标题（h1-h6）
- 树形展示层级结构
- 点击标题跳转到对应位置
- 高亮当前可视区域的标题
- 自动跟随滚动

## 数据流设计

### 状态管理

```javascript
AppState = {
  currentDirectory: FileSystemDirectoryHandle | null,
  currentFile: FileSystemFileHandle | null,
  files: FileItem[],
  headings: Heading[],
  isSidebarCollapsed: boolean,
  activeTab: 'files' | 'toc'
}
```

### 数据流图

```
用户操作                数据处理                    UI更新
────────────────────────────────────────────────────────────

打开目录 ──────────→ File System API ──────────→ 更新文件列表
                        │
                        ▼
选择文件 ──────────→ 读取文件内容 ──────────────→ 渲染Markdown
                        │
                        ▼
解析标题 ──────────→ 提取h1-h6 ────────────────→ 更新TOC
                        │
                        ▼
点击TOC ───────────→ 计算滚动位置 ──────────────→ 平滑滚动
```

### 关键数据流

1. **打开目录流程**
   ```
   用户点击"打开目录" 
   → 调用 showDirectoryPicker() 
   → 获取目录句柄 
   → 遍历目录读取所有.md文件 
   → 更新 files 状态 
   → 渲染文件列表
   ```

2. **选择文件流程**
   ```
   用户点击文件 
   → 获取文件句柄 
   → 调用 getFile() 读取内容 
   → marked.js 解析为HTML 
   → 更新 content 区域 
   → 提取标题生成TOC
   ```

3. **TOC跳转流程**
   ```
   用户点击TOC标题 
   → 获取标题元素ID 
   → 计算目标滚动位置 
   → 平滑滚动到目标位置 
   → 高亮当前标题
   ```

## 错误处理

### 1. 文件系统错误

| 错误类型 | 处理方式 |
|---------|---------|
| 用户取消选择 | 静默处理，不显示错误 |
| 权限被拒绝 | 显示友好提示，引导用户授权 |
| 文件不存在 | 显示"文件未找到"提示 |
| 文件读取失败 | 显示重试按钮 |

### 2. 解析错误

| 错误类型 | 处理方式 |
|---------|---------|
| Markdown解析失败 | 显示原始文本，标注错误位置 |
| 编码问题 | 尝试UTF-8，失败则提示用户 |

### 3. 网络错误（加载渲染库）

| 错误类型 | 处理方式 |
|---------|---------|
| CDN加载失败 | 使用本地备用库 |
| 超时 | 显示重试按钮 |
